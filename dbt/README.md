# Marketing Analytics dbt Project with MotherDuck

This dbt project provides analytics models for the marketing synthetic data generated by the parent project, using MotherDuck (cloud-based DuckDB) as the data warehouse.

## Prerequisites

- MotherDuck account and token
- AWS credentials (for S3 access to Parquet files)
- Python 3.8+
- dbt-duckdb installed

## Setup

### 1. Install Dependencies

From the root directory of the project:

```bash
# Create and activate virtual environment
python3 -m venv venv
source venv/bin/activate

# Upgrade pip and install all dependencies (including dbt-duckdb)
python3 -m pip install --upgrade pip
python3 -m pip install -r requirements.txt
```

### 2. Configure MotherDuck Connection

Copy the `profiles.yml` file to your dbt profiles directory:

```bash
# Create .dbt directory if it doesn't exist
mkdir -p ~/.dbt

# Copy the profiles file
cp dbt/profiles.yml ~/.dbt/profiles.yml
```

**Note:** The path `md:synthetic_data_lakehouse` tells dbt to connect to your `synthetic_data_lakehouse` database in MotherDuck. The profiles.yml is already configured to use environment variables for credentials.

### 3. Set Environment Variables

Create a `.env` file in the `dbt` directory from the example:

```bash
cd dbt
cp .env.example .env
```

Edit `.env` and add your credentials:

```bash
# AWS Configuration (for S3 access to Parquet files)
AWS_REGION=us-east-1
AWS_ACCESS_KEY_ID=your_aws_access_key_id
AWS_SECRET_ACCESS_KEY=your_aws_secret_key

# MotherDuck Token (get from https://motherduck.com)
MOTHERDUCK_TOKEN=your_motherduck_token_here
```

Load the environment variables:

```bash
source .env
```

Or export them directly:

```bash
export MOTHERDUCK_TOKEN=your_token
export AWS_REGION=us-east-1
export AWS_ACCESS_KEY_ID=your_access_key
export AWS_SECRET_ACCESS_KEY=your_secret_key
```

### 4. Test the Connection

```bash
cd marketing_analytics
dbt debug --profiles-dir ../
```

You should see output indicating a successful connection to MotherDuck and S3 access configured.

## Running dbt

### Run all models

```bash
cd marketing_analytics
dbt run --profiles-dir ../
```

### Run specific models

```bash
# Run only staging models
dbt run --select staging.* --profiles-dir ../

# Run only marts
dbt run --select marts.* --profiles-dir ../

# Run a specific model
dbt run --select  campaign_performance --profiles-dir ../
```

### Test data quality

```bash
dbt test --profiles-dir ../
```

### Generate documentation

```bash
dbt docs generate --profiles-dir ../
dbt docs serve --profiles-dir ../
```

## Models Overview

### Staging Models

These models clean and standardize the raw data:

- **stg_campaigns_daily**: Campaign performance with calculated CTR and CPC
- **stg_sessions**: Session data with engagement levels and page depth
- **stg_conversions**: Conversion data with value tiers
- **stg_attribution_touchpoints**: Attribution touchpoint data

### Mart Models (Fact Tables)

Business-ready analytics tables:

####  campaign_performance
Complete campaign performance metrics including:
- Spend, impressions, clicks, CTR, CPC
- Sessions, users, engagement
- Conversions, revenue
- Calculated metrics: conversion rate, ROAS, CPA

####  user_journey
User journey analysis including:
- Touchpoint count per user
- Campaign and channel diversity
- Journey length
- Conversion status
- User classification (multi-channel, converted, etc.)

####  channel_attribution
Multi-touch attribution by channel:
- First-touch attribution
- Last-touch attribution
- Linear attribution
- Revenue by attribution model

####  daily_summary
Daily rollup of all metrics:
- Active campaigns and channels
- Total spend, impressions, clicks
- Session and user metrics
- Overall conversion rate and ROAS

## Example Queries

After running `dbt run`, you can query the models in MotherDuck:

```sql
-- Top performing campaigns by ROAS
SELECT
    campaign_name,
    channel,
    total_spend,
    total_revenue,
    roas,
    conversion_rate
FROM marts. campaign_performance
WHERE roas > 1.5
ORDER BY roas DESC
LIMIT 10;

-- Channel attribution comparison
SELECT
    channel,
    first_touch_revenue,
    last_touch_revenue,
    linear_revenue,
    first_touch_conversions,
    last_touch_conversions
FROM marts. channel_attribution
ORDER BY linear_revenue DESC;

-- Daily performance trends
SELECT
    date,
    total_spend,
    total_revenue,
    overall_roas,
    overall_conversion_rate,
    total_sessions
FROM marts. daily_summary
ORDER BY date DESC
LIMIT 30;
```

## Development Workflow

1. **Make changes** to models in `models/`
2. **Run models**: `dbt run --profiles-dir ../`
3. **Test**: `dbt test --profiles-dir ../`
4. **Query results** in MotherDuck to validate
5. **Generate docs**: `dbt docs generate --profiles-dir ../`

## Targets

The project has three targets defined in `profiles.yml`:

- **prod** (default): MotherDuck connection, `main` schema
- **dev**: MotherDuck connection, `dev` schema
- **local**: Local DuckDB file for testing

To use a different target:

```bash
dbt run --target dev --profiles-dir ../
```

## Troubleshooting

### Connection Issues

If you get connection errors:

```bash
# Test MotherDuck connection
dbt debug --profiles-dir ../

# Verify environment variables are set
echo $MOTHERDUCK_TOKEN
echo $AWS_ACCESS_KEY_ID
```

### S3 Access Issues

Make sure your AWS credentials have read access to the S3 bucket with the Parquet files.

### Schema/Table Not Found

Verify the tables exist in MotherDuck:

```sql
SHOW TABLES;
```

If tables don't exist, you may need to create them first using the parent project's data generator.
